# UI Component Style

## Component Style Rules

Admin dashboard components should prioritize:

- readability
- predictable actions
- clear status
- safe destructive flows
- compact but not cramped layout

## Buttons

Use consistent placement:

- Primary action top-right of page header.
- Row actions in kebab/dropdown.
- Destructive action at bottom of detail or confirmation dialog.

## Cards

Use cards for:

- summary metrics
- system status
- detail group
- empty state

## Tables

Tables should support:

- search
- filter
- sort
- pagination/load more
- row action
- status badge

## Detail Drawers

Use detail drawers for fast inspection:

- order detail
- payment detail
- contact detail
- product quick view
- webhook event detail

## Modals

Use modals for:

- confirmation
- small create/edit forms
- high-risk actions
- payment manual override

## Toasts

Use toast for:

- saved successfully
- action failed
- copied link
- webhook set
- payment link created
