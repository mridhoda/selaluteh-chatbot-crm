# 0.5 Frontend Secret and Environment Guard

Task: `0.5 [P0] Define frontend secret and environment guard`

Date: 2026-07-08

Scope: Client environment audit and test-only guard. No runtime config changed.

## Audit

Client-exposed env names found in frontend source and env files:

- `VITE_API_BASE`: public API base URL. Public-safe if it points to the intended backend origin and never embeds credentials.
- `VITE_API_URL`: legacy/alternate public API base URL. Public-safe with same restrictions as `VITE_API_BASE`.
- `VITE_APP_NAME`: public display name. Public-safe.
- `VITE_DEMO_MODE`: public demo flag. Public-safe only for non-production/demo behavior.
- `VITE_PUBLIC_FILES_BASE_URL`: public file base URL. Public-safe if files served behind this URL are already intended for public/customer access.
- `VITE_DEV_API_PROXY_TARGET`: Vite dev server proxy target read by `vite.config.js` from Node `process.env`; local/dev only, not client-bundled if kept out of `import.meta.env` usage.
- `VITE_CLEANTER_ENABLED`, `VITE_CLEANTER_BASE_URL`, `VITE_CLEANTER_PRINT_PATH`, `VITE_CLEANTER_TIMEOUT_MS`, `VITE_CLEANTER_MAX_PAYLOAD_BYTES`, `VITE_CLEANTER_SUPPORTS_CUT`: public local thermal printing bridge configuration. Public-safe only as local client/device configuration, not provider credentials.

Forbidden in frontend env examples/source:

- Provider secrets: BayarGG, Xendit, DOKU, Midtrans, OAuth/client secrets.
- Webhook secrets/signing keys.
- Service-role keys.
- Private keys.
- Database passwords and connection strings.
- API keys/tokens that authorize privileged backend/provider access.

## Map

Classification:

- Environment audit: documentation-only.
- Static guard: greenfield Node test under existing `web/test` runner.

## Implementation

Added `web/test/phase-5/env-guard.test.mjs` to scan frontend source/env files for secret-shaped `VITE_*` names:

- `SECRET`
- `PRIVATE`
- `SERVICE_ROLE`
- `WEBHOOK`
- `TOKEN`
- `PASSWORD`
- `API_KEY`
- `ACCESS_KEY`
- `CLIENT_SECRET`
- `PROVIDER_SECRET`
- Payment provider secret patterns such as `BAYARGG_SECRET`, `XENDIT_SECRET`, and `DOKU_SECRET`

## Alpha Required Env

Minimum public-safe alpha env:

- `VITE_API_BASE` or `VITE_API_URL` pointing to the alpha backend.
- `VITE_APP_NAME` for display.
- `VITE_DEMO_MODE=false` for real alpha verification.
- Optional `VITE_PUBLIC_FILES_BASE_URL` only if customer-opened files are required.
- Optional Cleanter variables only for Android/local thermal printing tests.
