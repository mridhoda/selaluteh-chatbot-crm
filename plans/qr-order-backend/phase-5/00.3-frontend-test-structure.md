## Audit

Existing frontend test facts:

- Runner: `web/package.json` uses `node --test "test/**/*.test.mjs"`.
- Existing tests: `web/test/public-store-utils.test.mjs`, `web/test/file-access.test.mjs`, `web/test/thermal-print.test.mjs`.

## Map

## RED Test

Behavior-changing frontend tests were not added because this task prepares the harness and fixtures only.

Safety tests added:

- `contract-fixtures.test.mjs` verifies fixture request payloads avoid backend-owned authority fields.
- `env-guard.test.mjs` statically rejects secret-shaped `VITE_*` names in client source/env examples.

## GREEN Implementation

Added helpers:

- `renderPublicRoute(element, options)` for public route rendering through `MemoryRouter` and `renderToString`.
- `renderAdminRoute(element, options)` for admin route rendering fixtures without asserting auth authority.
- `createFakeApiResponse(data, status)` and `createFakeApiError(errorFixture, status)` in the fixture layer.
- `assertBackendErrorShape(error)`, `assertErrorState(result, expectedCode)`, and `assertNoFrontendAuthorityFields(payload)`.
- `createCheckoutIdempotencyKeyFixture(suffix)` and `createQrTokenFixture(suffix)`.
