# 0.6 Phase 5 Release Gates

Task: `0.6 [P0] Define Phase 5 release gates`

Date: 2026-07-08

Scope: Release gate definition for Phase 5 alpha. Documentation only.

## Blocking Checks

Required automated checks before alpha sign-off:

- `npm --prefix web run lint`: no new lint errors from Phase 5 changes.
- `npm --prefix web test`: Node unit/API-contract tests pass, including Phase 5 fixture/env guards.
- `npm --prefix web run build`: production frontend build succeeds.
- `npm run specs:check`: spec folder sync check passes.
- API client tests prove checkout sends `Idempotency-Key`, does not send `payment_status` or `fulfillment_status`, does not treat local totals as final, and never calls webhook endpoints.
- Component tests cover public storefront, QR resolve states, cart validation errors, checkout duplicate prevention, payment pending read-only status, public order privacy, and admin allowed actions.
- E2E/manual sandbox tests cover public online store, QR store, checkout/payment pending, public order tracking, admin order lifecycle, admin payment settings display, and existing CRM/chat/WhatsApp smoke flows.

## No-Go Conditions

Alpha is blocked if any of these are true:

- Frontend can mark payment as paid.
- Frontend sends `payment_status` or `fulfillment_status` in checkout/admin action requests.
- Frontend treats provider redirect/success return page as paid.
- Frontend uses local cart total as final authority.
- Frontend overrides QR locked outlet/location.
- Checkout can submit without an `Idempotency-Key`.
- Duplicate checkout can create duplicate visible orders/payments in sandbox.
- Admin UI guesses actions rather than rendering backend `allowed_actions`.
- Admin UI uses a generic status patch for Phase 5 order lifecycle instead of explicit endpoints.
- Public UI exposes raw provider payload, internal order IDs, webhook data, service-role details, or unmasked private customer fields.
- Existing admin/CRM/chat/WhatsApp flow has a blocking regression.
- Forbidden secret-shaped `VITE_*` env names are present in client source or deployed frontend env.

## Manual Sandbox Matrix

Public no-login flow:

- Open `/store/:storefrontSlug` on mobile and desktop.
- Select outlet when not QR-locked.
- Add available product and unavailable product edge case.
- Validate cart through backend and show backend validation errors.
- Submit checkout once, retry same attempt with same idempotency key, and start a new attempt with a new key.
- Confirm payment pending page remains read-only.
- Confirm payment success/cancel return does not mark paid until payment status endpoint says paid.
- Open public order tracking by opaque public token and verify masking/public-safe fields.

QR flow:

- Open valid outlet QR.
- Open expired/revoked/invalid QR.
- Confirm locked outlet/location cannot be changed client-side.
- Confirm QR session token is passed only where required by contract.

Admin flow:

- List orders, open detail, render backend `allowed_actions`.
- Accept, prepare, ready, complete, and cancel through explicit endpoints in allowed states.
- Confirm unpaid order cannot be accepted/prepared by UI or backend.
- Confirm admin payment settings render public-safe provider status without secrets.
- Confirm admin QR settings route/page remains hidden or disabled until backend contract exists.

Existing regression smoke:

- Login/logout.
- `/app/chats` list, send message, takeover/release where sandbox data allows.
- `/app/products` list and open product interactions.
- `/app/payments` list/detail.
- `/app/platforms` list.
- `/app/orders` existing legacy page behavior if still routed.

## Skipped/Flaky Policy

- A skipped P0 test requires a documented blocker, owner, and replacement manual check.
- Flaky P0 tests are release blockers until either fixed or explicitly downgraded with documented risk acceptance.
- No skipped test may hide an authority boundary: payment, fulfillment, QR lock, idempotency, allowed actions, secrets, or public privacy.

## Minimum P0 Criteria

Before alpha:

- Wave 0 audit/map/harness tasks complete.
- API client foundation and safety guards complete.
- Public storefront, QR, cart, checkout/payment pending, public tracking P0 flows complete.
- Admin order P0 list/detail/actions complete against explicit backend endpoints.
- Existing CRM/chat/WhatsApp/admin smoke checks recorded.
- All No-Go conditions cleared or documented as hard blockers.
