# Route

Public order tracking is available at:

```text
/order/:publicOrderToken
```

The legacy route remains available:

```text
/store/order/:publicOrderToken
```

The route is public and does not require admin login. It loads order data through `phase5ApiClient.public.getPublicOrder(publicOrderToken)`.

# Response Fields

`sanitizePublicOrder()` accepts backend public order response data and returns a safe UI model with:

- `publicOrderToken`
- `orderNumberPublic`
- `queueNumber`
- `publicOrderStatus`
- `paymentStatus`
- `fulfillmentStatus`
- masked customer name/phone fields
- outlet display data
- QR location label when present
- item snapshots
- backend totals
- sanitized public invoice links when returned

# Public Status Display

The order page displays:

- Safe public order number.
- Public order status timeline.
- Safe payment and fulfillment status labels.
- Item snapshot and backend totals.
- Outlet information.
- QR table/location label when returned by backend.

The model tests cover pending, failed, expired, and completed state normalization.

# Privacy Rules

The public order model strips fields that must not render publicly:

- Internal order IDs.
- Raw provider payloads.
- Admin notes.
- Admin user data.
- Sensitive customer phone/email values.
- Unsafe absolute invoice/provider links.

The public token is treated as an opaque route parameter and is never parsed for internal identity.

# Tests

- Payment pending/failed/expired/completed status modeling.

# Limitations

Invalid/expired token behavior is implemented through safe hook error handling, but browser DOM assertions are not available without a component runner. Full route rendering and mobile layout verification remain release-gate/manual checks for later Phase 5 stabilization.
