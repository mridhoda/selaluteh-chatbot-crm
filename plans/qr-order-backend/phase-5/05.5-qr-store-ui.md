# 05.5 QR Store UI

Tasks: `6.1` through `6.6`

Date: 2026-07-08

Scope: Public `/qr/:qrToken` route, QR resolve UI for Universal/Outlet/Location scopes, backend-derived QR locks, Node-testable QR model tests, and safety notes. Cart validation, checkout, payment, and order tracking remain Wave 3+.

## Route

Implemented route:

- `/qr/:qrToken` in `web/src/routes/privateRoutes.jsx`.

Route safety:

- The route is registered at the top level, before `/app/*`.
- It does not require admin login.
- It does not render `DashboardLayout` or admin navigation.
- The path `qrToken` is treated as opaque and is only sent to `phase5ApiClient.public.resolveQr(qrToken)`.

## QR Session State

Files changed for QR Store UI:

- `web/src/features/public-store/pages/QrStorePage.jsx`
- `web/src/features/public-store/index.js`
- `web/src/routes/privateRoutes.jsx`
- `web/src/features/public-store/components/OutletPickupBadge.jsx`
- `web/src/features/public-store/utils/publicStoreModel.js`
- `web/test/phase-5/qr-store-ui-model.test.mjs`

State model:

- `qrToken`: opaque route parameter used only for resolve.
- `qrSessionToken`: stored only after backend resolve response and included only in local intent context for future validation/checkout-related calls.
- `qrScope`: read from backend response fields (`qrScope`, `scope`, `qrCode.scope`, or session scope). If absent, lock fields are used only as fallback normalization.
- `lockedOutlet`: backend-provided outlet for Outlet/Location QR.
- `lockedLocation`: backend-provided location/table context for Location QR.
- `eligibleOutlets`: backend-provided eligible outlets for Universal QR.

Sensitive internals:

- No raw QR session internals are rendered.
- UI displays only public-safe scope, outlet name/address, and location/table label.
- QR session token is not displayed.

## Universal QR Behavior

Universal QR rules:

- Outlet selector is shown.
- Only backend-provided eligible outlets are selectable.
- Inactive/disabled outlets are filtered out.
- No outlet is auto-selected for Universal QR; the user must choose one before opening cart/checkout intent.
- Missing outlet shows an inline safe prompt: `Pilih outlet sebelum lanjut validasi atau checkout.`
- Selected outlet is cleared/reselected only through the backend-provided eligible list.

Safety:

- The frontend does not infer eligible outlets from token content.
- The frontend does not allow arbitrary outlet IDs outside the resolve response.

## Outlet QR Behavior

Outlet QR rules:

- Locked outlet is displayed from backend response.
- Outlet selector is disabled and not opened.
- Local override attempts are ignored in UI and rejected by `assertNoLockedQrOverride()`.
- Product availability is displayed in the locked outlet context.
- Future validation/checkout intent context includes `qrSessionToken` after backend resolve.

Safe errors:

- `QR_OUTLET_MISMATCH` maps to `This QR code is not valid for the selected outlet.`

## Location QR Behavior

Location QR rules:

- Locked outlet is displayed from backend response.
- Locked location/table label is displayed from backend response.
- Outlet selector is disabled.
- No location selector is rendered.
- Local location override attempts are rejected by `assertNoLockedQrOverride()`.
- Future validation/checkout intent context includes `qrSessionToken` and `qrLocationId` only after backend resolve.

Safe errors:

- `QR_LOCATION_MISMATCH` maps to `This QR code is not valid for the selected location.`

## Error Handling

Safe QR resolve states:

- Loading: existing `StoreSkeleton`.
- Invalid token: safe `StoreErrorState` fallback.
- Expired QR: mapped `QR_EXPIRED` message.
- Revoked QR: mapped `QR_REVOKED` message.
- Outlet mismatch: mapped `QR_OUTLET_MISMATCH` message.
- Location mismatch: mapped `QR_LOCATION_MISMATCH` message.
- No eligible outlet: safe `Outlet QR tidak tersedia` state.

No raw stack traces, raw backend internals, provider payloads, or admin information are rendered.

## Tests

Added Node-testable QR model tests:

- `web/test/phase-5/qr-store-ui-model.test.mjs`

Covered behavior:

- Universal QR success with backend-provided eligible outlet.
- Universal QR missing outlet remains unselected until a user chooses an eligible outlet.
- Outlet QR locked outlet UI model and override prevention.
- Location QR locked outlet/location model and override prevention.
- QR session token appears only in intent context after backend resolve.
- QR outlet/location mismatch safe error mapping.

Related public-store tests cover:

- Expired QR safe error mapping.
- Revoked QR safe error mapping.

Documented test limitation:

- Browser component tests for the actual `/qr/:qrToken` route rendering, modal interaction, and responsive controls were not added because no supported browser/component test runner exists.
- Current tests validate the pure QR authority and UI model consumed by `QrStorePage`.
